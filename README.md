# Workflow SI-2.0

Мишина Мария и Кондренко Дарья

## Описание

Автоматический процесс, который:
- Выполняется по расписанию (ежечасно)
- Получает данные о запчастях через HTTP GET с пагинацией
- Сохраняет данные в PostgreSQL в таблицк spare_parts
- Формирует CSV отчет и отправляет на внешний сервер
- Обрабатывает 10 записей на страницу

## Основные узлы

1. **Schedule Trigger** - запуск по расписанию
2. **HTTP GET** - получение данных из внешнего API
3. **UPSERT** - обновление/вставка в PostgreSQL
4. **Merge** - объединение данных
5. **Code (Page)** - расчет пагинации
6. **If** - проверка достижения лимита страниц
7. **SELECT** - выборка данных из БД для отчета
8. **Code (To csv)** - преобразование в CSV
9. **HTTP POST** - отправка CSV отчета

## Особенности

- Пагинация с автоматическим увеличением номера страницы
- Проверка на достижение максимума записей
- Конверсия данных в CSV формат
- Двусторонняя синхронизация с внешним API
- Логирование времени синхронизации
